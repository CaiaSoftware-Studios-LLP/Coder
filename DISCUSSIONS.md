# GitHub Discussions guide

**Caia Coder** is a **subscription product**. This repository does not contain the application source—it is where subscribers and the community **read docs**, **follow the changelog**, and **talk to each other** alongside [Issues](https://github.com/CaiaSoftware-Studios-LLP/Coder/issues).

Use [GitHub Discussions](https://github.com/CaiaSoftware-Studios-LLP/Coder/discussions) for conversation that does not need a tracked bug or a documentation pull request.

## When to use Discussions

| Category | Use for | Examples |
| -------- | ------- | -------- |
| **Q&A** | How-to questions, configuration help | “Best way to structure INTENT.md for a monorepo?” |
| **Ideas** | Product suggestions and workflows | “Export scheduler run history to CSV” |
| **General** | Tips, showcases, community chat | “Our team’s Kubernetes integration setup” |

## When to use Issues instead

Open an [Issue](https://github.com/CaiaSoftware-Studios-LLP/Coder/issues) when:

- Something is **broken** (bug report template).
- You want a **tracked feature** with acceptance criteria (feature request template).

## When to use email or the account portal

| Channel | Use for |
| ------- | ------- |
| [account.coder.caiaplatform.com](https://account.coder.caiaplatform.com) | Sign-up, plan changes, payment method |
| [billing@caiaplatform.com](mailto:billing@caiaplatform.com) | Invoices, refunds, plan disputes |
| [support@caiaplatform.com](mailto:support@caiaplatform.com) | Account access, enterprise inquiries |
| [security@caiaplatform.com](mailto:security@caiaplatform.com) | Vulnerabilities — see [SECURITY.md](SECURITY.md); do not post exploit details publicly |

## Before you post

1. Search existing Discussions and Issues for duplicates.
2. Include your **subscription context** when relevant: plan tier, SaaS vs self-hosted, browser/OS.
3. Do not post API keys, passwords, or customer data.
4. Follow the [Code of Conduct](CODE_OF_CONDUCT.md).
